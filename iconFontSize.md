# window桌面图标文本的显示规则

indows 桌面（本质上是一个名为 SysListView32 的控件）在渲染图标下方的文本时，遵循一套严格的底层排版和裁剪规则。这些规则由操作系统的 DPI 缩放、系统网格度量（WindowMetrics）以及 Explorer 的外壳渲染逻辑共同决定。

主要显示规则可以拆解为以下几个核心维度：
-----

## 1. 折行与截断规则（字数限制）

这是日常感知最明显的规则，Windows 根据图标的“聚焦状态”动态改变文本显示：
+ 常规（非聚焦）状态：
  + 最多显示 2 行文本。
  + 如果文本在当前的网格宽度内超过 2 行，第 2 行末尾会被强行截断，并显示省略号（...）。
  + 如果文本很短，只有 1 行，文本会自动居中显示。
+ 选中/聚焦状态：
  + 当用户鼠标悬停、点击或用方向键选中图标时，文本框会临时展开。
  + 最多可以展开显示 4 到 5 行（视具体系统版本而定），以便用户完整阅读长文件名。
  + 展开的文本框会暂时遮挡下方邻近网格的空白区域（但不会挡住下方图标的图片）。

## 2. 网格与间距限制（控制文本宽度）
文本能够显示多宽，完全取决于 Windows 桌面的隐藏网格大小。这两个关键数值存储在注册表中：
+ 注册表路径： HKEY_CURRENT_USER\Control Panel\Desktop\WindowMetrics

|注册表键名 |默认值 (100% DPI) | 含义|
|---|---|---|
|IconSpacing |-1125 (约 75 像素) | 图标的水平间距（包含文本的最大宽度边界）| 
|IconVerticalSpacing | -1125 (约 75 像素) |图标的垂直间距|

计算公式： 注册表中的数值是“缇”（Twips），像素值 = |数值| / 15。
文本的最大可用宽度即为 IconSpacing 的像素值减去两侧的留白（通常两侧各留 4-6 像素的安全距离）。一旦单个词或连续字符超过这个宽度，Windows 就会尝试强制换行。

## 3. 字体与 DPI 线性缩放
文本的字号并不是固定的，它会根据系统的 DPI 缩放和系统字体配置进行计算：

+ 默认字体： 简体中文 Windows 下默认为 Microsoft YaHei (微软雅黑)，英文下为 Segoe UI。
+ 默认字号： 在 100% 缩放（96 DPI）下，默认字号通常为 9磅 (9pt) 或 10磅。
+ DPI 随动： 当系统缩放变为 125%、150% 时，文本像素大小会严格按比例放大。例如：
   + 100% 缩放：9pt 字体 $\approx$ 12 像素高度。
   + 150% 缩放：字体实际渲染高度 $\approx$ 18 像素。
+ 字重： 桌面图标文本默认是 Regular（常规） 字重，不支持在未选中时自动变粗。

## 4.智能自适应换行（Word Wrapping）
Windows Shell 在处理文本换行时，使用的是标准的 Windows 文本路径截断算法（类似 DT_WORDBREAK 标签）：

+ 西方字符（带空格）： 如果是英文文件名，Windows 会优先在空格或连字符处换行，尽量保证单词的完整性。

+ 东方字符（中日韩）： 中文字符每个字都可以是换行点。如果一句话到边界了，哪怕是一个词语中间，也会直接硬换行。

+ 无空格长字符串： 如果是一个没有空格的超长英文单词或一连串数字（如 NewFiles20260518VersionFinalWithoutSpace），Windows 在无法容纳时会启动强行按字符换行，如果两行装不下，直接在第二行末尾加 ...。

## 5. 背景透明度与阴影（Legibility）
为了防止用户更换白色或复杂的壁纸后看不清图标文字，Windows 桌面具有以下保护机制：

+ Alpha 阴影（Drop Shadow）： 默认情况下，桌面文本是白色的，但带有动态的黑色半透明阴影。这样无论壁纸是黑是白，都能清晰分辨。

+ 性能回退： 如果用户在“高级系统设置 -> 性能”中关闭了“为桌面图标标签使用阴影”功能，文本背景将不再透明，而是变成一个块状的纯色矩形背景（通常为系统主题色或壁纸主色调的对比色），字体颜色也会随之调整。

## 开发人员查阅提示
如果你在写 C# 代码时需要获取文字实际占用的矩形大小，无法通过简单的 Graphics.MeasureString 完美模拟，因为 Explorer 有自己的外壳微调。最精准的方法是通过 P/Invoke 向桌面 SysListView32 发送 LVM_GETITEMRECT 消息，并将参数指定为 LVIR_LABEL，即可拿到当前每个图标文本在屏幕上的绝对像素坐标和宽高。
